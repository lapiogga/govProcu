## 시작하기

### 접속
- 데스크톱·태블릿 웹브라우저 (모바일 비대상)
- URL: `http://<서버주소>:3000`
- 로그인: 발급받은 Bearer 토큰 사용 (운영 환경에서만, PoC는 default)

### 핵심 가치
> 공고번호 또는 사업자번호 1개로 입찰의 전 생애 또는 업체의 시장 위치를 한 화면에서 추적

## 페르소나 1: 영업 매니저 (P1)

### 일일 루틴

**1. 신규 공고 다이제스트** (`/me`)
1. "키워드 알림 구독"에서 자사 분야 등록 (예: "정보화", "AI")
2. 다음날 오전 다이제스트로 신규 공고 자동 도착

**2. 관심 공고 깊이 분석** (`/bids/trace`)
1. 공고번호 입력 → 6단계 타임라인 즉시 표시
2. 응찰업체 표 → 누가 응찰했는지·얼마에 응찰했는지
3. 낙찰업체가 자사 경쟁사인지 확인 → 업체 프로필 클릭

**3. 경쟁사 분석** (`/vendors/[bizNo]`)
1. 경쟁사 사업자번호로 직접 진입 (또는 빠른검색)
2. 최근 6개월 낙찰 이력·평균가·낙찰률
3. 주요 거래 발주기관 Top 5

**4. 즐겨찾기 추적**
- 관심 공고는 `/me`에서 즐겨찾기로 등록
- 다음 방문 시 한 클릭으로 다시 추적

---

## 페르소나 2: 견적 담당 (P2)

### 응찰가 산정 시나리오

**1. 유사 사업 검색** (`/lookup?mode=bid` 또는 자연어 콘솔)
1. 기준 공고번호 입력
2. 같은 발주기관·같은 업종·같은 금액대 입찰 자동 매칭

**2. 발주기관 사정률 패턴** (`/agencies?name=...`)
1. 발주기관명 입력
2. 분위수 차트 (p10/p25/median/p75/p90) 확인
3. 자동 해석: "중낙찰률 (응찰가 88~92% 권장)"

**3. 투찰가 예측** (`/prediction`)
1. 공고번호 또는 기초금액+발주기관 입력
2. 목표 낙찰확률 선택 (0.7 = 상위 30% 응찰)
3. **추정 응찰가 + 95% CI** + 시나리오 6개 비교 차트

**4. 적격심사 점수** (`/qualification`)
1. 응찰가/기초금액/시공경험/기술자/신용등급 입력
2. 항목별 점수 + 총점 + 만점 대비 비율

## 페르소나 3: 임원/리더 (P3)

### 시장 위치 파악

**1. 자사 시장 점유** (`/analytics?type=용역`)
- 업종 선택 → Top 20 도넛 차트
- 자사 순위·점유율%
- 월별 동향 BarChart 2종

**2. 경쟁사 분석** (자연어 콘솔)
- "/console" 진입
- "1234567890 사업자, 최근 6개월 경쟁사 비교"
- LLM이 `vendor_profile` + `peer_analysis` 자동 호출

**3. 발주기관 의존도** (`/lookup?mode=biz`)
- 사업자번호 입력
- "주요 거래 발주기관" Top 10 분포

**4. KPI 대시보드** (`/`)
- 즐겨찾기 입찰 빠른 접근
- 빠른 검색 cmd+k 통합

---

## 페르소나 4: 분석가 (P4)

### 시장 리서치

**1. 발주기관 패턴** (`/agencies`)
- 사정률 분위수 차트
- 발주 이력 + 낙찰업체 매칭표 (수십 건)

**2. 업종 동향** (`/analytics`)
- 월별 BarChart (건수 + 합계)
- 시장 점유 도넛 (Top 5 + 기타)

**3. Cross-Lookup** (`/lookup`)
- xyflow 그래프로 4 키 관계 시각화
- 한 키 → 다른 3 키 자동 추적

**4. AI 자연어 콘솔** (`/console`)
- "공공기관 정보화 용역 시장 작년 동향"
- LLM이 적절한 도구를 자동 호출 (`industry_trend` + `market_share`)
- 결과를 `BidLifecycleCard` 등으로 즉시 시각화

## 빠른 검색 패턴

`/` 또는 cmd+k에서 입력:

| 입력 패턴 | 자동 라우팅 |
|---------|-----------|
| 10자리 숫자 | `/vendors/<bizNo>` (사업자번호) |
| 8~15자리 숫자 | `/bids/trace?no=<bidNo>` (공고번호) |
| 한글 + 키워드 | `/bids?q=<keyword>` (검색) |

## FAQ

### Q1. "추적이 5단계에서 멈춤"
- 사전규격은 일부 공고만 등록. 미등록은 정상.
- NTS 검증은 낙찰자 사업자번호가 응답에 있을 때만 동작.

### Q2. "투찰가 예측이 부정확"
- v0 휴리스틱: 발주기관 분위수 룩업
- v1 ML: 90일 학습 (CI 자동 갱신)
- 발주기관 샘플 5건 미만이면 업종 일반치 fallback

### Q3. "다중 발주기관 검색이 G2B만"
- LH/도로/수자원/코레일 어댑터는 키 발급 후 활성화
- `/agencies` 에서 `list_supported_agencies` 상태 확인

### Q4. "그래프 시각화 누락"
- `/lookup` 에서만 xyflow 활성 (Cross-Lookup)
- Neo4j R&D 트랙 통합 시 다른 페이지에도 확장 예정
