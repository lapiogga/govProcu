# GovProcu — 인터랙티브 프론트엔드 계획서 (UI-PLAN v1)

> 작성: 2026-05-02 00:55 KST
> 사용자 지시 (2026-05-02 00:53): "인터랙티브 프론트 환경 메뉴체계와 사용자 인터페이스 화면 계획 수립"

---

## 1. 비전

> **"공고번호 한 개 또는 사업자번호 한 개로 시작하면, 입찰 한 건의 전 생애 또는 한 업체의 시장 위치까지 한 화면에서 즉시 파악할 수 있는 인터랙티브 콘솔"**

> **타깃 환경 (사용자 5/2 01:31 KST 정정)**: **데스크톱·태블릿 웹앱 only**. 모바일 앱·PWA 비대상. 사무 환경에서 표·차트 풍부한 화면 우선.

MCP 서버에 노출된 42종 도구 중 **사용자 시나리오에 매핑된 핵심 도구**를 화면 단위로 묶고, 4개 핵심 relational key(`bid_notice_no` / `contract_no` / `inst_code` / `vendor_biz_no`)로 진입·이동이 가능하도록 설계한다.

---

## 2. 사용자 페르소나

| 페르소나 | 직책 | 핵심 KPI | 주요 화면 |
|---------|------|---------|----------|
| **P1 — 영업 매니저** | 입찰 영업 담당 | 신규 입찰 발견·낙찰률 | 입찰 검색·동향, 발주기관 발주이력, 경쟁사 분석 |
| **P2 — 견적 담당** | 응찰가 산정 | 적정 응찰가·낙찰률 | 유사 사업 검색, 시장 점유, 업종 동향 |
| **P3 — 임원/리더** | 의사결정 | 시장 위치·트렌드 | 대시보드, 업체 프로필, 시장 점유 |
| **P4 — 분석가** | 시장 리서치 | 통계·발주기관 패턴 | 발주기관 분석, 산업 트렌드, 통계 |

## 3. 메뉴 체계 (Information Architecture)

```
GovProcu 콘솔
├── 🏠 대시보드 (홈)
│   ├── 즐겨찾기 입찰
│   ├── 최근 본 항목
│   └── 주요 KPI 위젯 (이번 주 신규 공고/낙찰)
│
├── 🔍 입찰 검색
│   ├── 통합 검색 (search_bid_notices)
│   ├── 사전규격 (list_pre_specifications)
│   └── 낙찰 검색 (search_awards)
│
├── 📋 입찰 상세 추적 ⭐ (핵심 가치)
│   └── 공고번호로 진입 → trace_bid_lifecycle
│       ├── 사전규격 단계
│       ├── 본 공고 단계
│       ├── 개찰 + 응찰업체 단계
│       ├── 낙찰 단계
│       └── 낙찰업체 NTS 검증
│
├── 🏢 업체 분석
│   ├── 업체 프로필 (vendor_profile) — NTS + 입찰/응찰/개찰/낙찰 통합
│   ├── 사업자번호 검증 (verify_business_info)
│   ├── 다건 상태 조회 (check_business_status)
│   └── 업체 기준 검색 (V1~V4)
│       ├── 입찰 (search_bids_by_vendor)
│       ├── 응찰 (search_participations_by_vendor)
│       ├── 개찰 (search_openings_by_vendor)
│       └── 낙찰 (search_awards_by_vendor)
│
├── 🏛️ 발주기관 분석
│   ├── 기관 요약 (agency_bid_summary)
│   ├── 발주 이력 + 낙찰업체 (agency_procurement_history)
│   └── 발주규모 통계 (agency_procurement_volume)
│
├── 📊 분석/통계
│   ├── 동종업체 찾기 (find_similar_vendors)
│   ├── 유사 사업 찾기 (find_similar_bids)
│   ├── 업종 동향 (industry_trend)
│   ├── 경쟁사 분석 (peer_analysis / competitor_analysis)
│   ├── 시장 점유 (market_share / list_top_vendors_by_period)
│   └── 전체 조달 통계 (get_procurement_stats)
│
└── 🔗 Cross-Lookup (Relational Key)
    ├── 공고번호로 (lookup_by_bid_no)
    ├── 발주기관으로 (lookup_by_inst_code)
    ├── 사업자번호로 (lookup_by_biz_no)
    └── 계약번호로 (lookup_by_contract_no)
```

## 6. 기술 스택 제안

### 6.1 프론트엔드

| 영역 | 기술 | 이유 |
|------|------|------|
| 프레임워크 | **Next.js 15** (App Router) | RSC + 캐싱 + SEO |
| UI 라이브러리 | **shadcn/ui + Tailwind v4** | 디자인 시스템 + 다크모드 + 접근성 |
| 상태 | TanStack Query | MCP 응답 캐싱 + 재요청 |
| 차트 | **Recharts** | 동향·점유율 시각화 |
| 표/필터 | TanStack Table | 정렬·페이지네이션 |
| 인증 | Bearer Token (MCP_API_TOKENS) | 서버 측 검증 |

### 6.2 백엔드 통합

```
브라우저 ──HTTP──> Next.js Server Actions/Route Handlers
                  └──MCP HTTP transport──> GovProcu MCP Server
                                          ├── G2B (입찰/낙찰/계약/통계)
                                          └── NTS (사업자 검증)
```

- Next.js가 MCP HTTP 클라이언트 역할
- 응답을 type-safe한 React Server Components로 렌더링
- Server Actions로 검색 폼 제출

## 7. 사용자 시나리오 (UAT용)

### 시나리오 1 — 영업 매니저
1. 홈 → "정보화 용역" 검색 → 30건 결과
2. 한 공고 클릭 → 입찰 상세 추적 → 응찰업체 7개사 확인
3. 낙찰업체 클릭 → 업체 프로필 → 최근 6개월 낙찰률 33% 확인
4. "동종업체 비교" → 같은 규모 5개사 비교

### 시나리오 2 — 견적 담당
1. 홈 → 자기 회사 사업자번호 입력 → 업체 프로필
2. "유사 사업 찾기" → 비슷한 추정가/업종 입찰 10건
3. "업종 동향" → 월별 평균 낙찰률 추이 확인
4. 응찰가 산정에 활용

### 시나리오 3 — 임원
1. 홈 → "이번 주 KPI" 확인 (신규 124건 / 낙찰 87건 / 142억)
2. "시장 점유" → 자사가 정보화 용역 시장 N위 확인
3. "경쟁사 분석" → Top 5 경쟁사 낙찰합계 비교

## 8. 단계별 추진 계획

### Phase A — 골격 (1주)
- Next.js 15 프로젝트 부트스트랩
- MCP HTTP 클라이언트
- 기본 레이아웃 + 다크모드
- 인증 미들웨어

### Phase B — 핵심 3화면 (2주)
- 입찰 검색
- 입찰 상세 추적 (W1)
- 업체 프로필 (W2)

### Phase C — 분석 화면 (2주)
- 발주기관 분석
- 동종업체 비교
- 업종 동향 (Recharts)

### Phase D — Cross-Lookup + 대시보드 (1주)
- L1~L4 진입 화면
- 즐겨찾기·최근 본 항목 (localStorage)
- 통합 빠른 검색

### Phase E — 운영 + 시연 (1주)
- E2E 테스트 (Playwright)
- 시연 시나리오 검증
- 사용자 매뉴얼

총 7주 — MCP 서버 안정화와 병행 가능.

## 9. 성공 기준 (MVP 시점)

1. 사용자가 공고번호 1개를 입력하면 trace_bid_lifecycle이 5초 이내 6개 섹션 반환
2. 사용자가 사업자번호 1개를 입력하면 vendor_profile이 5초 이내 NTS+낙찰이력 반환
3. 4개 relational key로 진입한 화면에서 다른 키로 이동(클릭) 시 1클릭으로 전환
4. 데스크톱·태블릿(반응형)에서 핵심 3화면 동일 동작. 모바일 비대상.
