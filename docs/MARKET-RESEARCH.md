## 3. GovProcu에 빠진 핵심 기능 (우선순위)

### 필수 (시장 진입 자격)
1. **키워드/맞춤 알림 + 이메일·알림톡 푸시** — 모든 경쟁사 보유. 사용자 상태 저장 인프라 동시 구축
2. **즐겨찾기/관심 공고 워치리스트** — `add_to_watchlist`, `list_my_watchlist` 도구
3. **다중 발주기관 통합 검색** (LH·수자원·도로·코레일·국방조달) — 입찰나라·웰로비즈 표준
4. **적격심사 점수 계산기** — 건설/용역 1순위 활용. 산식만 구현

### 중요 (성과 차별화)
5. **사정률 빈도분석 / 발주처 예가패턴** — `analyze_agency_price_pattern` 신설
6. **AI 투찰가/낙찰 하한 예측** — 우리 historical 데이터 + ML
7. **공동도급(협정) 업체 매칭** — `find_consortium_partner`
8. **낙찰가 시각화 차트 페이로드** — Vega-Lite spec 반환

### 부가 (UX·확장)
9. **모바일 앱 / PWA**
10. **CSV/Excel/PDF 내보내기**
11. **입찰 일정 캘린더 (D-day)** — `get_bid_calendar`
12. **다국어 (영문·중문)** — 외자입찰
13. **결재 워크플로우 / 협업** — 댓글, 멤버 공유
14. **입찰내역서/계약내역서 자동생성**
15. **정부지원사업 통합** (R&D, 보조금)

## 4. GovProcu 차별화 강점 (확인 사실)

1. **MCP 서버 → LLM/Claude 자연어 직결** — 11개 경쟁사 중 **누구도** MCP 미보유 (전부 웹+모바일)
2. **Tier 2 통합 워크플로우 (W1~W5)** — `trace_bid_lifecycle`은 6단계 1콜. 경쟁사는 단계별 화면 이동 필요
3. **NTS 사업자 진위/휴폐업 통합** — 11개 중 **누구도 미보유**. B2B 신뢰도 검증 강점
4. **Relational Key Cross-Lookup** — 공고↔낙찰↔계약↔사업자 자동매칭. 경쟁사는 수동
5. **42종 도구의 조합성** — LLM이 자유 조합. SaaS는 정해진 화면
6. **오픈소스·자체호스팅** — 정부기관·대기업 보안망 내 배포 가능 (SaaS 거부 시장 흡수)

## 결론

**GovProcu는 MCP/LLM·NTS·통합 워크플로우·Cross-Lookup의 4개 영역에서 시장 유일 강점**을 보유. 다만 **알림·즐겨찾기·모바일 같은 시장 진입 표준이 빠져 있고**, 건설업 도메인 도구(적격심사·사정률·투찰가)는 인포21C/비드큐 대비 얕음.

**전략**: 
- (단기 P0) 알림·즐겨찾기·적격심사 = 시장 진입 자격 확보
- (중기 P1) 사정률·투찰가 ML·다중기관 = 건설업 도메인 깊이
- (장기 차별화) MCP·NTS·trace_bid_lifecycle·Cross-Lookup·오픈소스 = 한국 G2B 시장 선점
