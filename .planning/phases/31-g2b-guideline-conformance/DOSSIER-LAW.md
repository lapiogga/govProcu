## 0. 핵심 결론 (TL;DR)

1. **우리 frontend 라벨 다수가 법령 표준과 미세 불일치** — 가장 큰 문제 3개:
   - `/bids` 검색폼의 `발주기관` 라벨 — 법령상 정확한 명칭은 **"수요기관"** 또는 **"공고기관"**. G2B API 응답 필드도 `ntceInsttNm`(공고기관명) / `dminsttNm`(수요기관명)으로 분리되어 있음. 우리 코드는 둘을 혼동하여 단일 "발주기관"으로 표시.
   - `/qualification`의 **"입찰가격(bid_amount)" + "기초금액(base_amount)"** 입력 — 법령 표준은 **"투찰금액"** 또는 **"입찰금액"**과 **"기초금액"**. UI 라벨 "응찰가"는 비표준어 (G2B/법령 어디에도 없는 신조어). 정정 권고.
   - `/bids` 업종 옵션 `공사 / 용역 / 물품` — 법령 + G2B endpoint 분류는 **공사/용역/물품/외자(/기타)** 5종. "외자"·"기타" 누락은 **법령 표준 + G2B BidPublicInfoService 4 endpoint 모두 누락**. 외자(Frgcpt) 제외는 의도적이라면 명시 필요.

2. **법적 필수 입찰공고 항목 vs 우리 표시 항목** (국가계약법 시행령 제36조):
   - 법령 명시 의무 항목 9종 중 우리 frontend는 **3~5종만 표시** (공고일/공고제목/발주기관/추정가/마감일). **입찰참가자격 / 입찰서 제출 마감일시 / 개찰일시·장소 / 낙찰자결정방법 / 입찰방법 누락**.
   - W3 trace의 `Stage 2 본 공고` 단계에서 추정가만 노출 — 법령상 동일 단계 필수 정보(낙찰자결정방법, 입찰참가자격, 개찰일시 등)는 backend에는 있으나 frontend에 미노출.

3. **`/qualification` 점수 항목 라벨이 법령/세부기준과 부분 불일치**:
   - 우리: `시공경험 / 기술자 수 / 신용등급 / 경영·기타`
   - 법령(공사 적격심사 세부기준): **`해당공사 수행능력(시공경험·기술능력·시공평가결과 등)` + `입찰가격` + `경영상태` + `신인도`**
   - "기술자 수" 단독 입력은 **세부기준의 "기술능력" 평가에서 일부 항목**. 항목명 정정 + 평가 분야 보강 권고.

4. **W3 trace 6단계 라벨 — 법령 표준과 4단계는 일치, 1단계 명칭 보정 필요**:
   - "사전규격" → 정확한 명칭은 **"사전규격공개"** (행정안전부/조달청 공식 표기). 시행령 근거 별개 조항 아니나 정부입찰·계약 집행기준에 명시.
   - "본 공고" → **"입찰공고"** (시행령 제33조 표제어).
   - "개찰" → **"개찰"** (정확).
   - "낙찰" → **"낙찰자 결정"** (시행령 제42조 표제어).
   - "계약" → **"계약 체결"** (낙찰통지 후 10일 이내; 시행규칙 제48조).

## 1. 핵심 법령 정리

### 1.1 국가를 당사자로 하는 계약에 관한 법률 (국가계약법)

- **본법**: 국가법령정보센터 — [국가를 당사자로 하는 계약에 관한 법률](https://www.law.go.kr/lsInfoP.do?lsiSeq=277589)
- **시행령**: [국가를 당사자로 하는 계약에 관한 법률 시행령](https://www.law.go.kr/lsInfoP.do?lsId=002652&ancYnChk=0) (대통령령 제35453호, 2025-04-22 일부개정)
- **시행규칙**: [국가를 당사자로 하는 계약에 관한 법률 시행규칙](https://www.law.go.kr/lsLinkCommonInfo.do?lspttninfSeq=68537&chrClsCd=010202)

**입찰 관련 핵심 조항**:
- **시행령 제2조** — 정의 (추정가격, 예정가격 등 정의 — 정확한 원문은 law.go.kr 직접 확인 필요. 이하 §3에 정리)
- **시행령 제7조** — 추정가격 산정 기준
- **시행령 제8조** — 예정가격 작성 기준
- **시행령 제33조** — 입찰공고 (전자조달시스템 이용 의무, 정정 시 5일 이상 추가)
- **시행령 제35조** — 입찰공고의 시기 (추정가격 10억 미만 7일 / 50억 이상 40일 등)
- **시행령 제36조** — 입찰공고 명시사항 (필수 항목 9종 — §4 상세)
- **시행령 제42조** — 낙찰자 결정 (낙찰 결정 방법 — 적격심사·종합심사 등)
- **시행규칙 제48조** — 계약 체결 (낙찰통지 후 10일 이내)

**우리 코드 영향**:
- `/bids/trace` Stage 1~6 라벨이 시행령 표제어와 미세 불일치
- `/qualification` 입력 항목이 시행령 제42조에 따른 낙찰자 결정방법(=적격심사)의 세부 평가분야와 부분 불일치
- 입찰공고 표시 항목이 시행령 제36조 명시사항과 누락 다수

### 1.2 지방자치단체를 당사자로 하는 계약에 관한 법률 (지방계약법)

- **본법 + 시행령**: [지방자치단체를 당사자로 하는 계약에 관한 법률 시행령](https://www.law.go.kr/LSW/lumLsLinkPop.do?chrClsCd=010202&lspttninfSeq=81850) (대통령령 제35382호, 2025-03-12)
- **시행규칙**: [지방자치단체를 당사자로 하는 계약에 관한 법률 시행규칙](https://www.law.go.kr/lsInfoP.do?lsiSeq=196574) (행정안전부령 제599호, 2025-12-31)

**관계**:
- 국가계약법은 국가기관(본부/지방청/특별법인) 계약에 적용
- 지방계약법은 지방자치단체(시·도·군·구 + 산하 공사·공단) 계약에 적용
- 본 dossier는 두 법령이 일치하는 부분 위주로 정리 (용어/절차 99% 동일).
- 차이점: **낙찰자 결정 시 적격심사 통과 점수가 지방계약법 종합낙찰제 92점** (행정안전부 예규 — [지방자치단체 입찰시 낙찰자 결정기준](https://www.law.go.kr/%ED%96%89%EC%A0%95%EA%B7%9C%EC%B9%99/%EC%A7%80%EB%B0%A9%EC%9E%90%EC%B9%98%EB%8B%A8%EC%B2%B4%EC%9E%85%EC%B0%B0%EC%8B%9C%EB%82%99%EC%B0%B0%EC%9E%90%EA%B2%B0%EC%A0%95%EA%B8%B0%EC%A4%80))

### 1.3 조달사업법

- **법명**: 조달사업에 관한 법률
- 본 조사 범위에서는 입찰 절차 직접 규정 없음. 조달청의 수요기관 대행 조달계약 + 비축물자 등 규정.
- **우리 코드 영향**: 미미. 단, "수요기관"(조달청 외 위탁기관)과 "공고기관"(공고 발급 기관) 용어 정의는 본법에서 유래.

## 2. 용어 사전 — 법령 표준 vs 우리 frontend 라벨

| 우리 frontend 라벨 | 법령 표준 용어 | 정의 (요약) | 출처 (조항) | 정합 여부 |
|---|---|---|---|---|
| 추정가격 | **추정가격** | 예정가격 결정 전 산정한 가격. **부가가치세 제외**. 국제입찰 대상 판단 등에 사용 | 시행령 제2조 + 제7조 | ✅ 일치 |
| (없음) | **예정가격** | 입찰·수의계약 시 낙찰자/계약금액 결정 기준이 되는 가격. **부가가치세 포함**. 입찰 직전 비공개 작성·유지 | 시행령 제2조 + 제8조 | ⚠ 미사용 (W3 trace나 /qualification에 노출 안됨) |
| 기초금액 (`base_amount`) | **기초금액** | 시장조사 또는 원가계산 기반으로 산출 후 부가가치세 가산하여 작성한 가격. 예가요약서 형태. **공고 시 공개 가능** | 정부입찰·계약 집행기준 + 예정가격작성기준 | ✅ 일치 |
| (없음) | **복수예비가격** | 기초금액의 ±2% 또는 ±3% 내에서 작성한 15개 예가. 입찰 후 4개 추첨하여 평균이 예정가격 | 정부입찰·계약 집행기준 | ⚠ 우리 코드 미사용 (적격심사 점수 계산 시 필요할 가능성) |
| (없음) | **낙찰하한가** | 적격심사 통과 가능한 최저 입찰금액 = 예정가격 × 낙찰하한율 (공사 87.745% 등) | 시설공사 적격심사 세부기준 | ⚠ `/qualification`의 `lower_rate_pct`로 backend 산정 중이나 frontend 라벨 부재 |
| 응찰가 (`bid_amount`) | **입찰금액** 또는 **투찰금액** | 입찰자가 제출한 입찰서상의 금액 | 시행령 제42조 등 | ❌ "응찰가"는 법령 비표준어. **"입찰금액"** 또는 **"투찰금액"** 권고 |
| 입찰가 / 투찰가 | **입찰금액 / 투찰금액** | 동일 (구어). 법령은 "입찰금액", 실무 + G2B는 "투찰금액" 혼용 | — | ⚠ 통일 필요 |
| 낙찰가 / 낙찰금액 | **낙찰금액** | 낙찰자의 입찰금액 | 시행령 제42조 | ⚠ "낙찰가" 구어 — "낙찰금액" 권고 |
| 낙찰률 | **낙찰률** = 낙찰금액 ÷ 예정가격 (또는 추정가격) × 100 | 낙찰금액의 예정가격 대비 비율. **분모가 예정가격인지 추정가격인지 출처 따라 다름** | 실무 관행 (법령 명시 없음) | ⚠ 분모 명시 필요. G2B award 응답 `sucsfbidRate`는 보통 예정가격 기준 |
| 투찰률 | **투찰률** = 입찰금액 ÷ 예정가격 (또는 추정가격) × 100 | 입찰자가 써낸 가격의 예정가격 대비 비율 | 실무 관행 | ⚠ 우리 코드 미노출 |
| 발주기관 | **공고기관(`ntceInsttNm`)** + **수요기관(`dminsttNm`)** | **공고기관**: 입찰공고를 게시한 기관 (예: 조달청). **수요기관**: 실제 물품·용역·공사를 사용/요청한 기관 (예: 충남대학교). 조달청이 수요기관을 대신해 공고할 때 둘이 다름 | 조달사업법 + G2B API 명세 | ❌ 우리 코드는 둘 구분 없이 "발주기관" 1개 라벨 — **분리 표시 권고** |
| 공고일 | **입찰공고일** | 시행령 제35조 — 입찰서 제출 마감일 기준 N일 전에 공고해야 함 | 시행령 제35조 | ⚠ "공고일"보다 **"입찰공고일"** 권고 |
| 마감일 | **입찰서 제출 마감일시** | 입찰자가 입찰서를 제출할 수 있는 종료 시점 | 시행령 제36조 제1호 | ⚠ "마감일"은 너무 모호. **"입찰마감"** 또는 **"입찰서 제출 마감"** 권고 |
| 개찰 | **개찰** | 제출된 입찰서를 개봉하여 가격 + 자격을 검토하는 절차 | 시행령 제36조 + 제40조 | ✅ 일치 |
| 응찰 | **응찰** = 입찰참가 = 입찰서 제출 | 사용자가 입찰에 실제로 참가한 행위 | 실무 관행 (법령은 "입찰" 또는 "입찰참가") | ⚠ "응찰"은 구어. 단 G2B API 응답에 `participant`로 표기되어 일관성 위해 유지 가능 |
| 낙찰 | **낙찰자 결정** 또는 **낙찰** | 적격심사·종합심사 등을 거쳐 계약상대자로 선정 | 시행령 제42조 | ✅ 일치 |
| 계약 | **계약 체결** | 낙찰자가 낙찰통지 후 10일 이내 표준계약서로 체결 | 시행규칙 제48조 | ⚠ "계약" 단일어 — **"계약 체결"** 권고 |
| 사전규격 | **사전규격공개** | 입찰공고 전 구매규격을 나라장터에 공개. 2016-12-30부터 모든 물품·용역 의무 | 정부입찰·계약 집행기준 + [행정안전부 보도자료 2016-12-30](https://www.mois.go.kr/frt/bbs/type010/commonSelectBoardArticle.do?bbsId=BBSMSTR_000000000008&nttId=56735) | ⚠ "사전규격" → **"사전규격공개"** 권고 |
| 적격심사 | **적격심사** | 시공경험·기술능력·재무상태·과거 계약이행 성실도 종합 심사. 예정가격 이하 최저가 입찰자부터 순서대로 진행 | 시행령 제42조 + [국가 적격심사 세부기준 (조달청)](https://www.g2b.go.kr/pn/pnp/pnpe/UntyAtchFile/downloadFile.do?bidPbancNo=R25BK00898850&bidPbancOrd=000&fileType=&fileSeq=7) | ✅ 일치 |
| 종합심사낙찰제 | **종합심사낙찰제** | 입찰가격 + 공사수행능력 + 사회적책임 종합평가. 추정가격 100억 이상 공사 등 | 시행령 제42조의2 + 종합심사낙찰제 운영기준 | ✅ 일치 (단 우리 코드 미구현) |
| 일반경쟁 | **일반경쟁입찰** | 자격을 갖춘 모든 자에게 개방 | 시행령 제18조 | ⚠ "입찰" 접미 권고 |
| 제한경쟁 | **제한경쟁입찰** | 실적·도급한도액·시공능력·기술보유 등 객관적 기준으로 자격 제한 | 시행령 제21조 | ⚠ "입찰" 접미 권고 |

### 3.1 추정가격
> "**추정가격**이란 계약담당 공무원이 물품·공사·용역 등의 조달계약을 체결함에 있어 「국제입찰에 의하는 정부조달계약의 범위에 관한 고시」 또는 영 제4조 등에 의한 국제입찰 대상 판단 등의 기준으로 삼기 위해 예정가격이 결정되기 전에 영 제7조의 규정에 의해 산정된 가격을 말한다." (시행령 제2조)
- **부가가치세 제외**
- 출처: [추정가격 정의 — heiri.co.kr](http://www.heiri.co.kr/cost/cost.php?ID=02), [국가계약법 시행령 — yeslaw.com](http://www.yeslaw.com/lims/front/page/fulltext.html?action=LAW&lawid=1605)

### 3.2 예정가격
- 입찰 또는 수의계약 시 **낙찰자 및 계약금액 결정의 기준이 되는 가격**
- **부가가치세 포함**
- 입찰서 제출 마감 직전 비공개 작성 (복수예비가격 추첨 방식 등)
- 출처: [예가작성기준 — concm.net](http://www.concm.net/infomation19.htm), [예정가격작성기준 — law.go.kr 행정규칙](https://www.law.go.kr/%ED%96%89%EC%A0%95%EA%B7%9C%EC%B9%99/(%EA%B3%84%EC%95%BD%EC%98%88%EA%B7%9C)%EC%98%88%EC%A0%95%EA%B0%80%EA%B2%A9%EC%9E%91%EC%84%B1%EA%B8%B0%EC%A4%80)

### 3.3 기초금액
- 계약담당자가 시장조사 또는 원가계산으로 산정한 가격에 부가가치세를 가산한 가격
- **공고 시 공개 가능** (예가요약서 형태)
- 복수예비가격 작성의 기준 (기초금액 ±2% 또는 ±3%)
- 출처: [기초금액 vs 복수예비가격 vs 예정가격 — ppta.or.kr](https://ppta.or.kr/webzine/2021_01/g2.html), [기초금액·추정가격·예정가격 정리 — ubidders.com](https://ubidders.com/blog/page/13/기초금액,-추정가격,-추정금액,-예정가격,-복수예가,-낙찰하한가-등등-용어-정리/)

### 3.4 추정금액
- **추정가격 + 부가가치세 + 관급자재 비용** = 시공자가 받을 총금액

| 신용등급 | "경영상태" | ❌ 라벨 정정 — "신용등급"은 신용평가사 등급. 법령은 **"경영상태"** (재무비율 기반) |
| (없음) | 신인도 | ❌ 누락 |

### 7.1 단계별 표준 명칭 + 시기

### 7.2 추가 누락 단계 (법령상 존재)

- **현장설명** (공사 입찰 한정) — 입찰공고와 별도 단계. 시행령 제35조
- **입찰참가신청** — 입찰서 제출 전 자격 확인. 시행령 제36조
- **이의신청 / 입찰보증금 처리** — 시행령 제37조
- **계약 이행** + **준공/완료** + **하자보수** — 시행령 제64조 등

**우리 W3 trace는 6단계로 압축** — 의도적이라면 OK, 단 backend 도구가 추가 단계 정보를 갖는다면 frontend 노출 여부 검토 필요.

## 9. 결론 — Phase 31 추가 fix 항목 권고

> 우선순위: **P0 = 사용자 신뢰도 직결**, **P1 = 법령 정합 강화**, **P2 = 라벨 통일성**

### P0 (필수)

- **F24 (신규)**: `/bids` 업종 옵션에 **외자 추가** + (선택) 용역 세분화 옵션
  - G2B BidPublicInfoService Frgcpt endpoint 호출 누락 — 실제 외자 입찰이 검색에서 빠짐
- **F25 (신규)**: `/bids/trace` Stage 2 본 공고에 **입찰참가자격 + 낙찰자 결정방법 + 개찰 일시·장소 노출** (법령 의무)
  - backend `getBidNoticeDetail` 응답에 이미 존재 → frontend 미노출 문제
- **F26 (신규)**: `/bids` 결과 + `/bids/trace` Summary의 "발주기관" → **"공고기관 / 수요기관" 분리 표시**
  - G2B 응답 `ntceInsttNm` ≠ `dminsttNm`. 조달청 대행 공고 시 사용자 혼동

### P1 (강력 권고)

- **F27 (신규)**: `/qualification` 입력 라벨 정정
  - "응찰가" → **"입찰금액"** 또는 **"투찰금액"**
  - "기초금액" → **"예정가격"** (분모 명확화) — 또는 둘 다 입력받아 명시
  - "기술자 수" → **"보유 기술자 수"**
  - "신용등급" → 의도 명확화 (경영상태 또는 신인도 가점)
  - labelMap "기타" → **"신인도"**
- **F28 (신규)**: `/bids/trace` 6단계 라벨 정정
  - 사전규격 → **사전규격공개**
  - 본 공고 → **입찰공고**
  - 낙찰 → **낙찰자 결정**
  - 계약 → **계약 체결**

### P2 (가능하면)

- **F29 (신규)**: `/bids` 검색결과 컬럼명 정정
  - "공고일" → **"입찰공고일"**
  - "공고제목" → **"입찰공고명"**
  - "추정가" → **"추정가격"**
  - "마감일" → **"입찰마감"** 또는 **"제출마감"**
- **F30 (신규)**: 전 페이지 "낙찰가" / "응찰가" / "투찰가" 등 비표준어를 **"낙찰금액 / 입찰금액"**으로 통일
- **F31 (신규)**: `/qualification` 결과의 backend `note` 필드에 적격심사 세부기준 출처 명시 (예: "조달청 시설공사 적격심사 세부기준 2024-07-01 시행")
